```yaml

  datasource:
    url: jdbc:postgresql://localhost:5432/postgres
    username: postgres
    password: postgres
    driver-class-name: org.postgresql.Driver
  jpa:
    hibernate:
      ddl-auto: update

      hibernate:

      port: 6379
```


## How to Run

### 1. Start Infrastructure

A `docker-compose.yml` has been generated with all required services.

**Docker:**
```bash
docker-compose up -d
```

**Podman:**
```bash
podman-compose up -d
```

This will start:

- **k2r-rabbitmq** (rabbitmq:3-management) on port 5672
- **k2r-postgres** (postgres:latest) on port 5432
- **k2r-valkey** (valkey/valkey:latest) on port 6379

### 2. Run the Application

```bash
./mvnw spring-boot:run
```

### 3. Deploy to Cloud Foundry

A `manifest.yml` has been generated with service bindings for verification:\n\n```bash
# Build the project first
./mvnw package -DskipTests

# Push to CF

```

### 4. Key Notes

- **RabbitMQ**: Queues and exchanges are auto-created by Spring AMQP `@RabbitListener` annotations with `queuesToDeclare`. No manual queue setup is needed — Spring Boot auto-declares them on startup.
- **RabbitMQ Management UI**: http://localhost:15672 (guest/guest)
- **PostgreSQL**: Tables are auto-created by Hibernate with `ddl-auto: update`. The schema will be generated from your JPA `@Entity` classes on first startup.
- **PostgreSQL**: Connect via `psql -h localhost -U postgres -d postgres`
- **Valkey**: No pre-created data structures needed. Spring Data Redis auto-configures the connection and creates keys/caches on demand.
